## 项目概述

这是一个**矿山数据处理** Python 项目，解析来自矿山运营的各类 Excel 报表，包括：

- 柴油/燃油消耗数据
- 生产数据（白班/夜班）
- 电力消耗
- 工时/矿卡作业数据

## 现有命令

所有脚本均基于 CLI，接受输入文件路径作为主参数：

```bash
# 处理燃油/柴油数据
python excel_fuel.py <输入文件> [--year 年份]

# 处理生产数据
python excel_production.py <输入文件夹路径>

# 处理电力消耗
python excel_electrical.py <输入文件> [--year 年份]

# 处理工时/矿卡数据（支持文件或文件夹）
python excel_worktime.py <输入文件> [--year 年份] [--month 月份]
```

输出文件写入输入文件所在目录。

## 当前架构

项目包含 5 个独立的 Excel 处理脚本：

| 脚本 | 用途 | 输入格式 |
|------|------|---------|
| `excel_fuel.py` | 设备柴油消耗 | 包含"设备柴油消耗"的 Sheet |
| `excel_production.py` | 生产效率表 | Sheet 名称为数字（1-31） |
| `excel_electrical.py` | 电力消耗 | 包含"Electrical"的 Sheet |
| `excel_worktime.py` | 矿卡/挖机作业 | 白班/夜班 Excel 文件 |
| `excel_production_enhanced.py` | 增强版生产处理 | - |

所有脚本使用 **pandas** 解析 Excel，输出结构化数据，列包含：日期、班次（Day/Night）、设备名称及各类指标（工时、燃油、产量等）。

### 3. 设备台账表（待设计）

需要设计一个新的设备台账 Excel 模板，作为设备名称模糊匹配的标准库。

#### 核心字段

| 字段 | 说明 |
|------|------|
| 设备编号 | 唯一标识（如 EX-001） |
| 标准设备名称 | 规范化名称，用于系统内统一（如 NTE240） |
| 设备类型 | 矿卡/挖机/其他 |
| 额定装载量 | 单趟装载吨数 |
| 所属公司 | 承包商名称 |
| 备注 | 其他信息 |

#### 模糊匹配字段（用于兼容不规范的设备名称）

| 字段 | 说明 | 示例 |
|------|------|------|
| 别名1 | 常用缩写或俗称 | NTE |
| 别名2 | 另一种写法 | NTE240#1 |
| 别名3 | 包含编号的写法 | NTE240 #001 |
| 全称 | 含公司/车队前缀 | XX公司-NTE240 |
| 旧名称 | 历史曾用名称 | NTE240-旧版 |

> 注：模糊匹配时，系统会同时搜索"标准设备名称"和所有别名字段，支持前缀匹配、部分匹配和相似度匹配。

### 4. 设备名称模糊匹配

在各脚本中，设备名称格式不统一（如带编号、带空格），需要：
- 基于相似度算法（如 difflib / Levenshtein distance）
- 支持前缀匹配、部分匹配
- 匹配结果可追溯（记录原始名称 → 标准名称）

### 5. 多表联动

通过日期（索引）+ 设备名称（模糊匹配后）关联各表数据，生成综合统计报表。
