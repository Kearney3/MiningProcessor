# 工时年月选择与配置删除修复设计

## 背景
当前 GUI 中，工时数据处理已经在业务逻辑层支持 `year` 和 `month` 参数，但界面上没有对应输入控件，导致用户无法在图形界面中选择年月。与此同时，设备装载量配置区域虽然显示了复选框列，但“删除选中”功能无法稳定删除被勾选的行。

本次工作仅针对这两个问题做定点修复，保持现有 GUI 架构与交互风格不变。

## 目标
1. 在工时数据处理模块中增加年份和月份选择控件。
2. 年份、月份默认值使用当前系统年月。
3. 工时处理时将所选年月传递到既有处理逻辑中。
4. 修复设备装载量配置中“删除选中”无法使用的问题。
5. 保持现有页面布局、日志机制、配置保存方式不变。

## 非目标
1. 不重构整体 GUI 架构。
2. 不改变 `excel_worktime.py` 的处理逻辑与输出格式。
3. 不把配置表整体改造成独立 ViewModel 结构。
4. 不扩展新的配置导入/导出格式。

## 方案选择

### 方案 A（采用）
- 在工时模块新增 `Dropdown` 年份与月份选择控件。
- 默认值取当前系统年月。
- 保持 `gui/logic.py` 现有工时处理流程，只补齐 UI 与 refs。
- 为配置表的每一行显式绑定选中状态更新，保证“删除选中”依据真实勾选状态删除。
- 将新增、导入、加载配置三条路径统一到同一个配置行构造函数。

### 方案 B（未采用）
- 新增年月控件。
- 同时将配置表数据改造成独立数据模型，再统一渲染 `DataTable`。

未采用原因：改动范围过大，不符合本次“定点修复”的目标。

### 方案 C（未采用）
- 用 `TextField` 输入年月，并对删除逻辑做最小补丁。

未采用原因：输入更容易出错，用户体验和现有模块风格不一致。

## 详细设计

### 1. 工时模块界面调整
在 `gui/components.py` 的工时数据处理区域中新增两个下拉框：
- `year`: 年份选择
- `month`: 月份选择

默认值：
- 年份：当前系统年份
- 月份：当前系统月份

选项范围：
- 年份：与燃油/电力模块保持一致，提供有限年份范围
- 月份：`1` 到 `12`

工时模块布局调整为：
- 路径输入框
- 年份下拉
- 月份下拉
- 处理按钮

这样可以保持和其他处理模块一致的“路径 + 参数 + 处理”结构。

### 2. 工时处理数据流
`gui/logic.py` 已经具备以下行为：
- 从 `work_refs["year"].value` 读取年份
- 从 `work_refs["month"].value` 读取月份
- 传递给 `run_task(..., year=..., month=...)`
- 生成输出文件名 `YYYYMM_工作效率表.xlsx`

因此本次只需要保证：
- `create_modules_section()` 返回的 `module_refs["work"]` 中包含 `year` 与 `month`
- 下拉框值可以被正常读取并转换为整数

不改变现有后台线程处理、日志输出和文件命名逻辑。

### 3. 配置表删除选中修复
当前问题的根源是：
- 表格展示了复选框列
- 删除逻辑依赖 `row.selected`
- 但行对象没有通过统一方式可靠维护选中状态

修复方式：
1. 提取统一的配置行工厂函数，例如 `make_config_row(device, cap)`。
2. 在工厂函数中创建 `ft.DataRow` 时，显式绑定选中变化处理。
3. 当用户勾选/取消勾选时，将最新选中状态写回该行对象。
4. “删除选中”执行时，根据这些同步后的状态删除被选中的行。
5. 删除后重建表格并刷新页面。
6. `load_config()`、`add_device()`、`import_config()` 全部改为使用统一工厂函数，确保各种来源创建出的行行为一致。

### 4. 错误处理与兼容性
- 若当前系统年月超出年份下拉范围，则默认选用范围内最接近的值；优先保证界面可用。
- 若配置表中没有任何选中项，点击“删除选中”不报错，只刷新或保持现状。
- 若配置中的装载量不是合法整数，继续沿用当前保存时的日志警告行为。

### 5. 测试设计（TDD）
先写测试，再写实现。

#### 工时模块测试
目标：验证 UI 组装结果满足需求。

覆盖点：
1. `create_modules_section()` 返回的 `module_refs["work"]` 中包含 `year` 和 `month`。
2. 默认值为当前系统年月。
3. 月份选项覆盖 `1-12`。

#### 配置删除测试
目标：验证删除选中行为可用。

覆盖点：
1. 配置区创建出的行在标记为选中后，可以被删除。
2. 未选中的行会保留。
3. 不同来源（加载、新增、导入）创建的行遵循同一删除逻辑。

测试优先级：
1. 先补最小 pytest 测试文件。
2. 先让测试失败，确认覆盖的是缺失/错误行为。
3. 再做最小实现使其通过。

## 影响范围
预计改动文件：
- `gui/components.py`
- 可能新增一个与 GUI 相关的测试文件
- 如测试需要，可能少量调整 `gui/logic.py` 或辅助函数组织方式，但不改变外部行为

## 验收标准
1. GUI 工时模块可选择年份和月份，默认显示当前系统年月。
2. 点击工时“处理”时，所选年月能传入现有处理逻辑。
3. 设备装载量配置中勾选一行或多行后，点击“删除选中”可正确删除。
4. 新增、加载、导入产生的配置行都支持删除选中。
5. 针对上述行为的自动化测试存在，并且能通过。
